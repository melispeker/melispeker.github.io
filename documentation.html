① Binary Numbers
✮ Data can be represented using bits.
✮ A byte is a sequence of 8 bits.
✮ Binary is a number system that uses two digits. "1" represents an electrical signal in your computer being on. "0" represents an electrical signal in your computer being off.

② Data Compression
✮ Lossless data compression algorithms can usually reduce the number of bits stored or transmitted while guaranteeing complete reconstruction of the original data.
✮ Lossy data compression algorithms can significantly reduce the number of bits stored or transmitted but only allow reconstruction of an approximation of the original data.
✮ Data compression is used to save transmission time and storage space.

③ Extracting Information from Data
✮ Data sets pose challenges such as the need to clean data, incomplete data, invalid data, and the need to combine data sources.

④ Using Programs with Data
✮ Programs are used in an iterative and interactive way when processing information to allow users to gain insight and knowledge about data.
✮ Programmers can use programs to filter and clean digital data.

College Board AP Daily 3.1, 3.2, 3.3 Notes

① Variables and Assignments
✮ A variable is an abstraction inside a program that can hold a value.
✮ Some programming languages provide types to represent data, which are referenced using variables. These types include numbers, Booleans, lists, and strings.
✮ It is possible to change the value represented by a variable.
✮ The "←" operator can be used to assign a value to a variable.

② Data Abstraction
✮ An index is a common method for referencing the elements in a list or string using natural numbers.
✮ Data abstraction provides a separation between the abstract properties of a data type and the concrete details of its representation.
✮ Data abstraction can be created using lists, which allow bundling of variables like strings, numbers, etc.
✮ Developing a data abstraction in a program can result in a program that is easier to develop and maintain.

③ Mathematical Expressions
✮ An algorithm is a finite set of instructions that accomplish a specific task.
✮ Algorithms can be expressed in various ways, such as natural language, diagrams, and pseudocode.
✮ Every algorithm can be constructed using combinations of sequencing, selection, and iteration.

College Board AP Daily 3.4, 3.5, 3.6, 3.7, 3.8, 3.9, 3.10, 3.11, 3.12, 3.13 Notes

④ Strings
✮ Strings are sequences of characters used to represent text in programming.
✮ A substring is a part of an existing string.
✮ They are used for things like names, words, or sentences.
✮ You can combine, break apart, and search through strings.

⑤ Boolean Expressions
✮ A boolean is either true or false.
✮ It can have a value of either "1" or "0"
✮ It has operators like AND, OR, and NOT.
✮ They are commonly used in conditionals and loops.

⑥ Conditionals
✮ Conditionals are statements in programming that allow the code to make decisions based on certain conditions.
✮ They use if, else if, and else for different codes depending on whether a given Boolean expression is true or false.
✮ For example: -if a condition is true, do something. -else if another condition is true, do something else. -else do something if none of the conditions are true.

⑦ Nested Conditionals
✮ Nested conditionals are conditionals within other conditionals.
✮ This means that an if, else if, or else statement contains another conditional inside it.
✮ They are used to handle more complex decision-making processes

⑧ Itearation
✮ Iteration is a repeating portion of an algorithm. Iteration repeats a specified number of times or until a given condition is met.
✮ Iteration statements change the sequential flow of control by repeating a set of statements zero or more times, until a stopping condition is met.
✮ In REPEAT UNTIL(condition) iteration, is the condition evaluates to true initially, the loop body is not executed at all, due to the condition being checkers before the loop.

⑨ Developing Algorithms
✮ Algorithms can be created from an idea, by combining existing algorithms, or by modifying.

⑩ Lists
✮ For list operation, write expressions that use list indexing and list procedures.
✮ For algorithms involving elements of a list, write iteration statements to traverse a list.

⑪ Binary Search
✮ Binary search efficiently finds a target in a sorted list by repeatedly dividing it in half.
✮ It compares the target to the middle element and searches left or right based on the comparison.
✮ Stops when the target is found or no more divisions are possible.
✮ Works only on sorted lists

⑫ Calling Procedures
✮ Procedures are reusable blocks of code for specific tasks.
✮ Calling a procedure runs its code, often with input parameters.
✮ Procedures can return values for further use.
✮ They reduce repetition and simplify code.

⑬ Developing Procedures
✮ Procedures solve tasks by breaking problems into smaller parts.
✮ Parameters and clear input/output make them flexible.
✮ Test procedures to ensure correctness and handle edge cases.
✮ Use comments and documentation for clarity and understanding.

College Board AP Daily 4.1, 4.2, 4.3 Notes

① The Internet
✮ The internet is a global network that connects computing devices using standard protocols.
✮ Data is transmitted in small chunks called packets, which travel independently and may take different paths.
✮ The Transmission Control Protocol (TCP) ensures reliable data transmission by checking for errors and ensuring packets are received in the correct order.
✮ The Internet Protocol (IP) assigns unique addresses to devices and routes data between them.

✮ The Domain Name System (DNS) translates human-readable domain names (e.g., google.com) into IP addresses.

② Fault Tolerance
✮ Fault tolerance is the ability of a system to continue operating despite failures in some components.
✮ The internet is designed to be fault-tolerant by routing data dynamically through alternative paths if a connection fails.
✮ Redundancy in network infrastructure improves reliability and minimizes the impact of failures.

③ Parallel and Distributed Computing
✮ Parallel computing divides a task into smaller subtasks that run simultaneously on multiple processors, improving efficiency.
✮ Distributed computing involves multiple computers working together to solve a problem, often over a network.

College Board AP Daily 5.1, 5.2, 5.3, 5.4, 5.5, 5.6 Notes

① Beneficial and Harmful Effects
✮ Computing has both positive and negative impacts on individuals and society.
✮ Beneficial effects include increased access to information, automation of tasks, improved communication, and advances in medicine.
✮ Harmful effects include privacy concerns, job displacement due to automation, misinformation, and negative effects on mental health.

② Digital Divide
✮ The digital divide refers to the gap between those who have access to computing technology and those who do not.
✮ It can be caused by economic, geographic, or social factors.
✮ A lack of access to technology can limit education, job opportunities, and participation in the digital economy.
✮ Efforts to bridge the digital divide include government policies, nonprofit initiatives, and infrastructure development.

③ Computing Bias
✮ Computing bias occurs when algorithms or data favor certain groups unfairly.
✮ Examples include facial recognition systems that perform poorly on certain demographics and biased hiring algorithms.
✮ Mitigating bias requires diverse datasets, algorithm transparency, and ethical considerations in software design.

④ Crowdsourcing
✮ Crowdsourcing uses the collective intelligence of a large group to solve problems or gather information.
✮ Examples include Wikipedia, citizen science projects, and open-source software development.
✮ Crowdsourcing can improve efficiency, reduce costs, and leverage diverse perspectives. However, it may also lead to misinformation if not properly monitored.

⑤ Legal and Ethical Concerns
✮ Legal and ethical concerns in computing include data privacy, intellectual property, and cybersecurity.
✮ Ethical dilemmas arise in areas like AI decision-making, data collection, and software piracy.
✮ Companies and individuals must follow ethical guidelines to ensure responsible technology use.

⑥ Safe Computing
✮ Safe computing practices protect personal data and online security.
✮ Best practices include using strong passwords, enabling two-factor authentication, and avoiding suspicious links.
